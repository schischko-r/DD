# DD-Индекс

Локальный набор файлов для витрины DD-индекса. Основная актуальная страница:

`DD-Индекс - титульная страница_v2.html`

Она загружает данные из `dd-data.json`, считает DD-индекс по продуктам и строит титульную таблицу без моковых данных.

## Файлы

- `dd-data.json` — источник строк с метриками.
- `DD-Индекс - титульная страница_v2.html` — титульная страница на реальных данных из JSON.
- `DD-Индекс - титульная страница.html` — исходная версия страницы, оставлена без изменений.
- `Agent.html` — страница детального анализа, используется как ориентир по загрузке данных.
- `1.ipynb` — рабочий ноутбук для быстрых проверок расчетов.

## Запуск

Открывать страницу лучше через локальный HTTP-сервер, потому что браузер может заблокировать загрузку `dd-data.json` при открытии HTML как обычного файла.

```bash
python3 -m http.server 8765
```

После запуска открыть:

```text
http://localhost:8765/DD-%D0%98%D0%BD%D0%B4%D0%B5%D0%BA%D1%81%20-%20%D1%82%D0%B8%D1%82%D1%83%D0%BB%D1%8C%D0%BD%D0%B0%D1%8F%20%D1%81%D1%82%D1%80%D0%B0%D0%BD%D0%B8%D1%86%D0%B0_v2.html
```

## Формат данных

`dd-data.json` ожидается как массив строк. Используемые поля:

- `unit` — юнит продукта.
- `product_group_uuid` — идентификатор продуктовой группы.
- `product_name` — название продукта.
- `period` — период расчета.
- `metric` — код метрики.
- `value` — набранное значение метрики.
- `max_value` — максимальное значение метрики.
- `is_metric_applicabble_flg` — признак применимости метрики.

Для продуктов без `unit` на странице показывается группа `Без юнита`.

## Расчет DD

Страница автоматически берет самый свежий период из JSON.

Формула DD по продукту:

```text
floor(sum(value) / sum(max_value) * 100)
```

Правила расчета:

- метрики с `auto_regularity` не участвуют в индексе;
- неприменимые метрики не добавляются в знаменатель;
- метрики с `max_value <= 0` считаются серыми и не добавляются в знаменатель;
- средний DD в шапке считается как среднее значение DD по продуктам.

## Светофоры

Светофоры строятся по реальным метрикам:

- зеленый — `value == max_value`;
- желтый — `0 < value < max_value`;
- красный — `value == 0`;
- серый — метрика неприменима или `max_value <= 0`.

Порядок отображения: зеленые, желтые, красные, серые.

## Контрольные значения

Для текущего `dd-data.json` должны сходиться такие значения:

- строк: `1368`;
- продуктов: `38`;
- юнитов: `4`;
- актуальный период: `II кв. 2026`;
- средний DD: `45%`.

Продукты `ВЗР (выезжающие зарубеж)` и `Вклады` должны попадать в группу `Без юнита`.

## Поведение интерфейса

- Доступны режимы сортировки `По юнитам` и `По DD-индексу`.
- Дельта кв/кв показывается только если в JSON есть предыдущий период.
- При одном периоде фейковая динамика не рисуется.
- Кнопка действия в строке продукта называется `Перейти →`.
- Если `dd-data.json` недоступен или пуст, страница показывает ошибочное состояние.
