# Переход рейтинга Data Driven на flat_table.xlsx

Дата: 2026-07-14
Ветка: `codex/gravity-ui`

## Что изменено

- `flat_table.xlsx` стал основным и дефолтным источником для `build_calc_report.py` и `build_gravity_report.py`.
- Одна книга формирует и титульный рейтинг, и детальные блоки продукта.
- Поле `flg` стало единственным правилом участия строки в Data Driven Index:
  - `flg = 1` — строка входит в числитель и знаменатель рейтинга;
  - `flg = 0` — строка отображается, но имеет `excluded_from_index = true`.
- Удалено дополнительное исключение A/B-тестов и других метрик по жестко заданным кодам для нового flat-table-пайплайна.
- Строки с одинаковыми `metric_name`, но разными `metric_code`, больше не склеиваются. Техническое различение не попадает в пользовательские названия.
- 88 текстовых строк «Потребности» без `metric_group` сохранены как отдельный витринный блок; текст из ненумерованного значения показывается в описании метрики.
- `СХ` нормализуется в `CX`.
- Каналы сохраняют исходный юнит `DP` или `PC`; искусственный юнит `Каналы` не создается.
- До появления колонки `type` тип временно выводится из `Column4`:
  - встречается `канал` — `Канал`;
  - `Column4` полностью пуст — `Сегмент`;
  - иначе — `продукт`.
- Явное непустое поле `type`, когда оно появится в книге, имеет приоритет над временным выводом.
- Windows-сборка теперь находит `npm.cmd`, поэтому `build_gravity_report.py` снова выполняет полный Vite-build.

## Профиль источника

- Лист: `main`.
- Строк: 4 018.
- Содержательных строк метрик: 4 016; две строки без имени метрики и `flg` пропускаются.
- Сущностей: 95.
- Типы: 81 продукт, 7 каналов, 7 сегментов.
- Юниты после нормализации: `CBP`, `CX`, `DB`, `DP`, `PC`, `ДомКлик`, `УБ`.
- Исходные строки `flg = 1`: 3 484.
- Исходные строки `flg = 0`: 532; все 532 присутствуют в детализации и исключены из рейтинга.

## Формула

Для каждой сущности:

```text
Data Driven Index = round(
  100 * SUM(факт, где flg = 1) / SUM(макс балл, где flg = 1)
)
```

Строки с нечисловыми значениями сохраняются для отображения, но не увеличивают числитель или знаменатель.
